# Реализация аудио плеера и улучшение навигации

## Обзор изменений

### 1. Постоянная загрузка данных для главных экранов

**Проблема:** Данные терялись при переключении между экранами главной и профиля.

**Решение:**
- Обновлен `HomeScreen` для использования `AppStateService` вместо прямого вызова `FirebaseService`
- Добавлен `StreamSubscription` для отслеживания изменений пользовательских данных в реальном времени
- Реализована правильная очистка подписок в `dispose()`
- `DashboardScreen` и `ProfileScreen` уже использовали `AppStateService`

**Результат:** Данные теперь постоянно загружаются и не теряются при навигации.

### 2. Профессиональный аудио плеер для подкастов

**Проблема:** Отсутствовал полнофункциональный плеер для воспроизведения подкастов.

**Решение:**

#### AudioPlayerService (Singleton)
- Управление глобальным состоянием аудио плеера
- Поддержка плейлистов с автоматическим переходом к следующему треку
- Потоковое обновление состояния воспроизведения, позиции и длительности
- Методы: `play()`, `pause()`, `resume()`, `stop()`, `seekTo()`, `playNext()`, `playPrevious()`
- Форматирование времени в формат MM:SS

#### AudioPlayerWidget
- Красивый градиентный дизайн с Material Design
- Элементы управления: play/pause, next, previous, seek slider
- Отображение обложки, названия трека и исполнителя
- Визуальный прогресс воспроизведения с интерактивным слайдером
- Кнопка закрытия плеера
- Адаптивный дизайн для разных размеров экрана

#### Интеграция в приложение
- Глобальное размещение `AudioPlayerWidget` в `MainScreen`
- Доступ к плееру с любого экрана приложения
- Обновлен `PodcastsScreen` для использования нового сервиса
- Сохранение состояния воспроизведения при переключении между экранами

### 3. Улучшение навигации

**Изменения в MainScreen:**
- Добавлен новый экран `HomeScreen` в навигацию
- Обновлена структура навигации: Дом → Главная → Помодоро → Эйзенхауэр → ДЗ → Расписание → Подкасты → Профиль
- Горизонтальная прокрутка для всех элементов навигации
- Адаптивные размеры для разных экранов

### 4. Технические улучшения

**Firebase Realtime Database:**
- Все данные хранятся в пути `users/{uid}/` для каждого пользователя
- AppStateService управляет подписками на изменения данных
- Кэширование данных для предотвращения потерь

**Архитектура:**
- Синглтоны для глобальных сервисов (`AppStateService`, `AudioPlayerService`)
- Потоковая архитектура для реактивных обновлений UI
- Правильное управление жизненным циклом ресурсов

**UI/UX:**
- Профессиональный градиентный дизайн
- Плавные анимации и переходы
- Интуитивные элементы управления
- Визуальная обратная связь для всех действий

## Использование

### Воспроизведение подкастов:
1. Перейдите на экран "Подкасты"
2. Нажмите на трек для воспроизведения
3. Используйте глобальный плеер внизу экрана для управления
4. Плеер остается доступным при переключении между экранами

### Постоянная загрузка данных:
- Данные пользователя и метрики теперь постоянно обновляются
- Информация сохраняется при навигации между экранами
- Автоматическое восстановление при потере соединения
